# Quick Start

## Cloud Deploy (Railway)

The fastest way to get started — one click, zero configuration:

[![Deploy on Railway](https://railway.com/button.svg)](https://railway.com/deploy/pv3JyA?referralCode=EqdcFH)

Railway auto-provisions PostgreSQL, wires all environment variables, and deploys all 3 services. After deployment, sign up on the dashboard and add your provider API keys (LiveKit, Google, etc.).

---

## Local Development (Docker Compose)

All 4 services start with one command.

## Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed and running
- Provider keys (LiveKit, Google Gemini, Resemble AI, STT) are configured via the dashboard after first boot

## 1. Clone the Repository

```bash
git clone https://github.com/Arkenos-World/Arkenos.git
cd Arkenos
```

## 2. Configure Environment Variables

```bash
cp .env.example .env
cp backend/.env.example backend/.env
cp frontend/.env.example frontend/.env.local
```

Edit each file and add your credentials. See [Environment Variables](/configuration/environment-variables) for the full reference.

<Note>
The agent needs **no `.env` file** — it fetches all API keys from the backend dashboard on startup. Auth is handled by Better Auth using your Postgres database — no external auth keys needed. The root `.env` only needs `POSTGRES_PASSWORD`.
</Note>

## 3. Run

```bash
docker-compose up -d --build
```

This starts 5 services:

| Service | URL |
|---------|-----|
| Frontend | http://localhost:4200 |
| Backend API | http://localhost:4201 |
| Voice Agent | Connected to LiveKit |
| PostgreSQL | localhost:5434 |
| MinIO | http://localhost:9002 |

## 4. Open the Dashboard

Visit [http://localhost:4200](http://localhost:4200) and sign in.

## 5. Configure API Keys

Go to **API Keys** in the dashboard and add your provider credentials:

1. **LiveKit** — API Key, API Secret, Server URL (from [LiveKit Cloud](https://cloud.livekit.io))
2. **Google** — API Key (for Gemini LLM)
3. **Resemble AI** — API Key (for TTS)
4. **STT Provider** — At least one of: AssemblyAI, ElevenLabs, or Deepgram

Use the **Test Connection** button on each card to verify. The agent service will pick up the keys automatically on its next restart.

## Useful Commands

```bash
# View logs
docker-compose logs -f

# View logs for a specific service
docker-compose logs -f backend

# Stop all services
docker-compose down

# Stop and remove volumes (resets database)
docker-compose down -v

# Rebuild a single service
docker-compose up -d --build backend
```
