# Quick Start (Docker Compose)

The easiest way to run Arkenos. All 4 services start with one command.

## Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed and running
- API keys for: Clerk (frontend auth), plus provider keys (LiveKit, Google Gemini, Resemble AI, STT) configured via the dashboard after first boot

## 1. Clone the Repository

```bash
git clone https://github.com/dhruv0206/Arkenos.git
cd Arkenos
```

## 2. Configure Environment Variables

```bash
cp backend/.env.example backend/.env
cp frontend/.env.example frontend/.env.local
```

Edit each file and add your credentials. See [Environment Variables](/configuration/environment-variables) for the full reference.

<Note>
The agent needs **no `.env` file** — it fetches all API keys from the backend dashboard on startup. Only `backend/.env` (Postgres password) and `frontend/.env.local` (Clerk + API URL) are required.
</Note>

## 3. Run

```bash
docker-compose up -d --build
```

This starts 4 services:

| Service | URL |
|---------|-----|
| Frontend | http://localhost:3000 |
| Backend API | http://localhost:8000 |
| Voice Agent | Connected to LiveKit |
| PostgreSQL | localhost:5432 |

## 4. Open the Dashboard

Visit [http://localhost:3000](http://localhost:3000) and sign in with Clerk.

## 5. Configure API Keys

Go to **Settings → API Keys** in the dashboard and add your provider credentials:

1. **LiveKit** — API Key, API Secret, Server URL (from [LiveKit Cloud](https://cloud.livekit.io))
2. **Google** — API Key (for Gemini LLM)
3. **Resemble AI** — API Key (for TTS)
4. **STT Provider** — At least one of: AssemblyAI, ElevenLabs, or Deepgram

Use the **Test Connection** button on each card to verify. The agent service will pick up the keys automatically on its next restart.

## Useful Commands

```bash
# View logs
docker-compose logs -f

# View logs for a specific service
docker-compose logs -f backend

# Stop all services
docker-compose down

# Stop and remove volumes (resets database)
docker-compose down -v

# Rebuild a single service
docker-compose up -d --build backend
```
